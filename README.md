# BankApp

Микросервисное банковское приложение на Spring Boot.  
Проект реализует регистрацию и авторизацию пользователей, работу со счетами, пополнение и снятие средств, переводы между своими счетами и переводы другим пользователям, а также просмотр курсов валют.

## Функциональность

В приложении реализованы следующие возможности:

- регистрация нового пользователя;
- вход в систему через Spring Security;
- просмотр данных текущего пользователя;
- просмотр списка счетов пользователя;
- открытие нового счёта в выбранной валюте;
- пополнение счёта;
- снятие средств со счёта;
- перевод между своими счетами;
- перевод другому пользователю;
- просмотр текущих курсов валют;
- межсервисное взаимодействие через OAuth2 и Keycloak.

## Архитектура проекта

Проект построен в виде набора микросервисов.

### Сервисы

- **front-ui** — пользовательский веб-интерфейс на Spring MVC + Thymeleaf;
- **accounts-service** — управление пользователями и банковскими счетами;
- **cash-service** — пополнение и снятие средств;
- **transfer-service** — переводы между счетами;
- **exchange-service** — работа с курсами валют и конвертацией;
- **exchange-generator-service** — генерация и обновление курсов валют;
- **blocker-service** — сервис проверок и блокировок операций;
- **notifications-service** — отправка уведомлений;
- **config-server** — сервер централизованной конфигурации;
- **discovery-server** — Eureka Server для сервис-дискавери;
- **postgres** — база данных PostgreSQL;
- **keycloak** — сервер авторизации OAuth2 / OpenID Connect.

## Безопасность

В проекте реализованы два уровня безопасности.

### Пользовательская аутентификация

Во `front-ui` реализован вход пользователя через Spring Security.  
Пользователь вводит логин и пароль, после чего получает доступ к личному кабинету.

### Межсервисная авторизация

Для взаимодействия между сервисами используется OAuth2 с `client_credentials` и Keycloak.

Пример сценария:

1. `front-ui` получает access token в Keycloak;
2. с этим токеном обращается в `accounts-service`;
3. `accounts-service` проверяет JWT как resource server.

Аналогично защищены основные межсервисные вызовы между сервисами.

## Основные пользовательские сценарии

После запуска приложения пользователь может:

1. зарегистрироваться;
2. войти в систему;
3. открыть один или несколько счетов;
4. пополнить счёт;
5. снять средства;
6. перевести деньги между своими счетами;
7. перевести деньги другому пользователю;
8. посмотреть курсы валют на главной странице.

## Запуск проекта

### 1. Требования

Для запуска должны быть установлены:

- Java 21
- Maven
- Docker
- Docker Compose

### 2. Сборка проекта

Из корня проекта выполнить:

```bash
mvn clean install -DskipTests
```

### 3. Запуск проекта через Docker Compose

```bash
docker compose up --build -d
```

### 4. Просмотр логов

```bash
docker compose logs -f
```

Логи конкретных сервисов:

```bash
docker compose logs -f front-ui accounts-service cash-service transfer-service exchange-service blocker-service notifications-service
```
